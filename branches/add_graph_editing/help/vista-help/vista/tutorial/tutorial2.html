Chapter 2




Sorting

In the Group Frame click on the table headers labeled A-F to sort alphabetically by those names. Clicking again toggles between ascending and descending orders. All sorting is stable and thus clicking on B part followed by C part ensures that for the same C part the B part sort order will be honored.

Filtering

In the Group Frame go to the section "Filter". Enter regular expressions ( or just plain old characters ) for any but atleast one of the parts A-F. Then click on the "Enter" key to either select or reject the matching data set names. One can also enter a regular expression for the Pathname Filter field in the same section. This filters on the entire pathname as opposed to the parts which filter based on part name matches.

Time Series Arithmetic

In the Group Frame go to the section "Math". Select the "Math" tab. Click on one of the pathnames in the table to select it. Then click on the operation such as +,-,* and / buttons in the Math section. Then click on another pathname followed by clicking on the "=" button in the Math section. This will create a new pathname next to the selected pathname. This is a new data set that is the result of the operation -> First selection ("+" or "-" or "*" or "/") second selection. The new pathname only exists in memory and needs to be explicitly saved to a dss file.

To use a number as one of the operands select the checkbox "Use Number" in the Math section. Then enter the number for the operation. The select the pathname and click the operation to be performed on it. This will create a new pathname next to the selected one which is the result of that operation

Complex operations can be built by operating on two pathnames or a pathname and a number at a time multiple number of times.

Setting Time Window

In the Group Frame go to the bottom of the window where there is a "Set TimeWindow:" followed by a textfield where a phrase may be entered. First select the item or items in the table for which you want to set the time window. The enter a time window in the following format:-

ddMMMyyyy Hhmm - ddMMMyyyy Hhmm

where dd is day's e.g. 02

where MMM is month, e.g. JAN

where yyyy is year, e.g. 1980

An example would be: 01JAN1994 0300 - 09JUN1996 2400

Time Series Operations

These operations are scattered in the Math Panel under the "Period Ops","Filling" and "Averaging" tabs.

Merging: Select the pathnames to be merged. Also reorder the references in the order of merging from top to bottom with top being given the highest priority. Reordering may be done by holding the shift button when dragging a row to the appropriate position. Then press the "Merge References" button in the Math Section, "Period Ops" tab. This will create a merged pathname which contains the first acceptable value from the selected pathnames in their priority ordering.

Period Averaging, Maximum and Minimum: Select the pathnames to be period averaged. Select the "Period Ops" tab in the Math section of the Group Frame. Select from the rightmost combobox an appropriate time interval to which period averaging is required. Also select the appropriate type of period operation from the next left combobox and then click on the "Do Operation" button. This will create new pathname(s) which will be the period average of the pathnames selected.

Moving Average: Select the pathname to be moving averaged. Select the "Averaging" tab in the Math section. Input an integer value for the number of next points to be included in the forward length and the number of previous points to be included in the backward length fields. Then click the "Do Average" button. This will create a new pathname with the moving average.
